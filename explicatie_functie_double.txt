Programul AtomC modelat in genTestProgramDouble:

f(2.0);

void f(double n){
	double i=0.0;
	while(i<n){
		put_d(i);
		i=i+0.5;
	}
}

Apelul functiei in codul MV:

OP_PUSH_F 2.0
	Pune pe stiva argumentul functiei, adica valoarea double n=2.0.

OP_CALL adresa_f
	Pune pe stiva adresa de revenire, apoi sare la prima instructiune a functiei f.

OP_HALT
	Este instructiunea la care se revine dupa terminarea functiei f.

Cadrul functiei:

n[-2] ret[-1] oldFP[0] i[1]

n[-2]
	Parametrul primit de functie. Pentru apelul f(2.0), aici se afla valoarea 2.0.

ret[-1]
	Adresa de revenire salvata de OP_CALL. Dupa terminarea functiei, executia continua de aici.

oldFP[0]
	Valoarea vechiului FP, salvata de OP_ENTER, ca sa poata fi restaurata la iesirea din functie.

i[1]
	Variabila locala double i. OP_ENTER 1 rezerva o celula pentru ea, apoi codul pune 0.0 in FP[1].

Ce este linia LESS.i din exemplul initial:

000001A211A52080/6 LESS.i // 1<2 -> 1

000001A211A52080
	Este adresa din memorie a instructiunii curente. Aceasta adresa poate fi diferita la fiecare rulare.

/6
	Arata ca in acel moment sunt 6 valori pe stiva. Inainte de LESS.i au fost puse pe stiva cele doua valori care se compara, peste cadrul functiei.

LESS.i
	Compara doua valori int scoase de pe stiva. In exemplu compara 1 si 2.

// 1<2 -> 1
	Comentariul arata calculul efectiv. Deoarece 1<2 este adevarat, rezultatul depus inapoi pe stiva este 1.

Pentru functia cu double trebuie aceeasi idee, dar cu instructiunea pentru double:

000001A211A52080/6 LESS.f // 1.5<2 -> 1

In genTestProgramDouble, conditia while(i<n) este generata astfel:

OP_FPLOAD 1
	Pune pe stiva variabila locala i.

OP_FPLOAD -2
	Pune pe stiva parametrul n.

OP_LESS_F
	Compara valorile double i si n si pune pe stiva un int: 1 daca i<n, 0 daca i>=n.

OP_JF dupa_while
	Foloseste rezultatul produs de LESS.f. Daca rezultatul este 0, sare dupa while.

Ce returneaza genTestProgramDouble:

Functia returneaza un pointer Instr* catre prima instructiune din lista de cod MV generata.
In cazul acesta, prima instructiune este OP_PUSH_F 2.0.
Acest pointer este dat mai departe catre run(testCode), iar run executa instructiunile in ordine.

Ce inseamna RET_VOID 1:

RET_VOID arata ca functia nu returneaza o valoare, fiind void.
Argumentul 1 inseamna ca functia are un singur parametru pe stiva: n.
La executie, RET_VOID 1 restaureaza IP din ret[-1], restaureaza FP din oldFP[0] si coboara SP astfel incat parametrul, adresa de revenire, vechiul FP si variabila locala sa fie eliminate din cadrul functiei.
